# Сводка по проекту Flyntos

**Flyntos** — это современная модульная платформа (монорепозиторий) для поиска авиабилетов с поддержкой локализации, построенная на базе современного веб-стека.

---

## 🛠 Технологический стек

### Основные технологии и инструменты:
- **Управление зависимостями**: `pnpm` (рабочие пространства / workspaces), Node.js (версия 22+).
- **Среда выполнения**: Node.js.
- **Язык программирования**: TypeScript (с общей конфигурацией в `tsconfig.base.json`).
- **Инфраструктура и развертывание**: Docker Compose, Docker (отдельные Dockerfile для каждого сервиса в директории `docker/`).

---

## 📂 Структура проекта (Монорепозиторий)

### Сервисы (`apps/`):
1. **`apps/web`** — Веб-интерфейс на **Next.js 15** и **React 19**.
   - Использует Tailwind CSS для стилизации.
   - Поддерживает динамическую маршрутизацию с учетом локали (`[locale]`).
   - Содержит переработанную главную страницу в премиальном темном стиле с эффектом Glassmorphism (эффект матового стекла), радиальным свечением на фоне и кастомным переключателем языков.
2. **`apps/api`** — Бэкенд API на **Fastify 5**.
   - Маршруты: `/search`, `/health`, `/config`, `/providers`.
   - Использует **Zod** для валидации контрактов и входящих запросов.
3. **`apps/worker`** — Фоновый воркер на базе **BullMQ**.
   - На данный момент зарегистрированы очереди и настроено базовое логирование (заготовка).

### Общие пакеты (`packages/`):
- **`packages/config`** — Конфигурация локалей, сценариев поиска и очередей.
- **`packages/i18n`** — Ресурсы локализации (словарь переводов).
- **`packages/provider-sdk`** — SDK для интеграции с провайдерами поиска.
- **`packages/search-contracts`** — Общие типы данных и схемы валидации (канонический контракт поиска).
- **`packages/seo-engine`** — Поисковая оптимизация.
- **`packages/shared`** — Утилиты и вспомогательные функции общего назначения.
- **`packages/ui`** — Общая библиотека UI-компонентов.

## 🚀 Что уже сделано и работает

1. **Базовая инфраструктура монорепозитория**:
   - Настроен запуск всех сервисов через единый менеджер пакетов (`pnpm dev`) или контейнеризацию (`docker compose up --build`).
   - Налажена типизация и линтинг во всех частях проекта.

2. **Интерактивный интерфейс поиска**:
   - Пользователь может ввести данные в форму поиска на главной странице (`apps/web/src/components/search/search-form.tsx`).
   - При отправке формы происходит переход на страницу результатов `/:locale/results`, где данные формы преобразуются в поисковый запрос к API.
   - Отображаются карточки результатов билетов (`ResultCard`) с фильтрацией на лету.

3. **Бэкенд-валидация и обработка поиска**:
   - API принимает POST-запросы на `/search`, валидирует их с помощью схемы Zod (`searchInputSchema`).
   - Происходит нормализация данных, после чего вызываются заглушки (mock) провайдеров авиабилетов.
   - Настроена ранжирование результатов (`rank-results.ts`) и фильтрация результатов на стороне сервера (`filter-results.ts`).

4. **Премиальный дизайн и локализация**:
   - Реализована адаптивная главная страница с современным визуальным стилем.
   - Создан удобный единый переключатель локалей в шапке (`locale-switcher.tsx`).
   - Подключена интернационализация через `packages/i18n`.

## ⚠️ Текущие ограничения и зоны роста (Что не сделано)

- **Реальные интеграции**: Вся выдача билетов пока построена на моковых провайдерах (`mock-search.ts`). Нет интеграции с реальными GDS/API авиакомпаний.
- **База данных**: База данных PostgreSQL описана в `docker-compose.yml`, но пока никак не задействована в коде.
- **Авторизация**: Отсутствует система регистрации и входа пользователей.
- **Тесты**: В проекте на текущий момент нет юнит- или интеграционных тестов.
- **Воркеры**: `apps/worker` пока выполняет лишь роль заготовки и не обрабатывает реальные фоновые задачи.
